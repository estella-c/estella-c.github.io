Heuristic Review and UX Testing

Overview

Roles: UX/UI Research, Project Management


Problem: A nationwide service company was updating their employee route-scheduling system. Management and the development team wanted feedback from users to evaluate their current prototype. The prototype system aimed to address several pain-points such as inefficient and repetitive user-flow, as well as creating an easier-to-use user interface.

What I did: I familarized myself with the system and current pain points through conducting usability tests and a user interview. I then completed an independent heuristic review. My observations and recommendations were summarized in a presentation for management and the development team. Since completing this review of the prototype system, the system has been updated.

Reflection: Overall, I improved my skills in usability testing and heuristic reviewing as I had to quickly familarize myself with the needs of several different teams and then map each teams' needs to possible (and realistic) solutions. Some challenges I faced included:
  • Prototype: The prototype system is not final, meaning it requires consistent user evaluation to continue improvement. My recommendations had to be reevaluated as changes were being made to the system.
  • Features: Many features were implemented by the development team without user input. I learned that while there are general features that can be implemented successfully without user evaluation, most features need to be implemented carefully according to user input to be useful to users.

To help align product development and user needs, we conducted a heuristic review and a few usability tests on the current route-planning prototype.

We first conducted usability tests with some employees: Users were asked to schedule a route for the day using the prototype scheduling system. As we observed the users, we took note of any comments or frustrations. After the usability tests, I conducted an interview with one of the users, a manager, to get a better idea of user pain-points. Following this, I conducted an indepenndent heuristic review evaluating the overall layout of the site. Overall:

  • Usability Tests: Users reported some improvements but still encountered frustrations with the layout and certain features.
  • User Interview: I got more insight on the issues the employees percieved most critical like issues with filtering locations and scheduling locations at specific times.
  • Heuristic review: Elements such as the right-to-left layout, the filtering tab, and method of scheduling time-specific locaitons could be improved upon.


A. Color Coding

Color coding and user feedback.

According to the responses received, color coding is unclear to the users.

    Recommendations:
  1. Color legend: Clearly label colors (currently, colors represent last-visited date).
  2. Misattributed colors: Some labels of "unattended" properties were marked in red, when they should really match the status of the managing property.
  3. Time intervals: One user reported uncertainty with the time scale (recency) represented by the colors. Could be clarified in the key or in the overlay.
  4. Overall, users reported positive feelings towards the direction of the color-coded points.


B. Work Flow

Work flow could be improved.

Workflow is currently from right to left.

    Recommendations:
  1. Right to Left: Currently, workflow begins with filters on the left, to the map, then to the route items.
  2. Date and Time: Could these be updated to better align with opening hours?


C. Icons

Saving and Loading feature not functioning as expected.

Users were generally unclear and very distracted by the meaning of some of the icons or buttons present on the screen.

    Recommendations:
  1. Unclear functions: Save the data as clearly labeled widgets or on a separate tab on the left-most wing.
  2. Bookmarks: Separate tab for bookmarks, use a bookmark icon, eye for overlay feature. Users are unclear on where to locate and modify bookmarked locations.
  3. One user suggestion: Allowing multiple "saved" routes to be compared or overlaid. Maybe having a tab with all saved routes and multiple "layers" with visibility toggles can address this.


D. Further Recommendations

QoL: Scheduling and Filters.

Additional features, including our recommendations and improvements requested by users.

    Recommendations:
  1. User feedback: Allow scheduling calender to pull from existing scheduled jobs.
  2. Hours: Allow users to view location hours or modify hours.
  3. Filtering: Implementing clear filters or search.


Next Steps

Updated System UI.

In response to our analysis, developers updated the route planning system with several of our suggestions to better address user needs.


Results: An improved map scheduling system was completed. Particularly:

  • Updated color coding with a key.
  • A clear scheduling feature with a time-block visualization.
  • Implementation of a filtering system: Allowing users to quickly narrow down locations.


Next steps: Evaluating other elements of the system redesign and discussing overall system improvements. Usability tests and wireframes may also be needed.
